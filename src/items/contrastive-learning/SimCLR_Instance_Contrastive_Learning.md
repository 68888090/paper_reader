# SimCLR: A Simple Framework for Contrastive Learning of Visual Representations

## 基本信息

- **论文标题**: A Simple Framework for Contrastive Learning of Visual Representations
- **作者**: Ting Chen, Simon Kornblith, Mohammad Norouzi, Geoffrey Hinton（Google Research, Brain Team）
- **发表时间**: 2020年（ICML 2020, Proceedings of the 37th International Conference on Machine Learning, Vienna, Austria）
- **刊物**: PMLR 119, 2020. arXiv:2002.05709v3 [cs.LG]
- **代码**: https://github.com/google-research/simclr

## 面向的问题

学习无需人工标注的有效视觉表示是一个长期难题。此前的主流方法分为两类：

1. **生成式方法**（如 VAE, GAN）：建模像素级别的输入分布，但像素级生成计算代价高昂，对表示学习而言可能并非必需。
2. **判别式方法**：使用启发式设计的代理任务（pretext tasks），如图片旋转预测、拼图排列等，但启发式设计可能限制学习表示的通用性。

本文关注的是**基于对比学习的判别式方法**。当时的对比学习方法（如 MoCo、PIRL、CPC）虽然取得了不错的成绩，但通常依赖**专门的架构**或**记忆库（memory bank）**等复杂组件，方法碎片化，缺乏系统性理解。SimCLR 的目标是：**在不使用专门架构或记忆库的前提下，简化对比学习框架，并系统性研究哪些组件使得对比学习有效**。

## 核心方法定位：实例级对比学习

SimCLR 是一种**实例级（instance-level）对比学习**方法。其核心思想是：

> 在表示空间中**最大化同一实例的不同增强视图之间的一致性**，同时将不同实例的特征推开。

这意味着：
- **正样本**：同一张图片的两个不同数据增强版本（来自**同一实例**）
- **负样本**：Mini-batch 中所有其他图片的增强版本（来自**不同实例**）
- **对比粒度**：以**整张图片（实例）**为单位——模型学习区分"这张图 vs 那张图"，而非关注图片内部的局部区域

这种实例级的判别目标迫使模型学习到**对语义变化敏感、但对数据增强不变**的特征表示。

## 方法流程

### 第一步：数据增强模块（前处理——关键创新之一）

SimCLR 系统研究了数据增强的组合对对比学习的影响，发现**数据增强的组成是定义有效预测任务的关键**。

使用的三种增强操作（顺序执行）：
1. **随机裁剪（Random Crop）**：随机裁剪一个区域，然后 resize 回原始大小
2. **随机颜色失真（Random Color Distortion）**：包含颜色抖动（color jittering）、颜色丢弃（color dropping）等
3. **随机高斯模糊（Random Gaussian Blur）**

**关键发现**：
- 仅使用随机裁剪不够——没有颜色失真时，模型会利用颜色直方图作为"捷径"来区分裁剪区域，而非学习语义特征
- **随机裁剪 + 颜色失真的组合**至关重要——裁剪迫使模型学习局部区域到全局视图的对应，颜色失真防止模型依赖简单的颜色统计
- 无监督对比学习需要比监督学习**更强的数据增强**

对每张输入图片 $x_k$，采样两次独立的增强操作 $t \sim \mathcal{T}$ 和 $t' \sim \mathcal{T}$，生成两个增强视图 $\tilde{x}_{2k-1} = t(x_k)$ 和 $\tilde{x}_{2k} = t'(x_k)$，构成**正样本对**。

### 第二步：Base Encoder（前向传播）

使用一个**基础编码器** $f(\cdot)$ 从增强数据中提取表示向量：

$$h_i = f(\tilde{x}_i) = \text{ResNet}(\tilde{x}_i)$$

- 采用标准 ResNet（如 ResNet-50），无任何架构约束
- $h_i \in \mathbb{R}^d$ 是平均池化层后的输出（如 2048 维）
- 实验证明更宽更深的网络带来更好的表示质量

### 第三步：Projection Head（非线性投影——关键创新之二）

引入一个小的神经网络**投影头** $g(\cdot)$，将表示映射到应用对比损失的空间：

$$z_i = g(h_i) = W^{(2)}\sigma(W^{(1)}h_i)$$

- $g(\cdot)$ 是一个含单隐藏层的 MLP
- $\sigma$ 为 ReLU 非线性激活
- $z_i$ 为用于对比损失的投影向量（如 128 维）

**关键发现**：
- 在 $z_i$ 上（而非 $h_i$ 上）计算对比损失效果显著更好
- **训练完成后丢弃 $g(\cdot)$**，仅使用 $f(\cdot)$ 输出的 $h$ 用于下游任务
- 原因：$h$ 保留了更多信息（如颜色、方向等），而 $z$ 被训练为对数据增强不变——这导致 $z$ 丢失了部分对下游任务有价值的信息
- 非线性投影头使 $h$ 和 $z$ 之间可以学习一个"信息过滤"变换

### 第四步：对比损失函数（前向 + 反向传播）

使用 **NT-Xent（Normalized Temperature-scaled Cross Entropy Loss）**损失函数。

对于一个包含 $N$ 个原始样本的 mini-batch，经数据增强后得到 $2N$ 个数据点。对于一个正样本对 $(i, j)$，损失定义为：

$$\ell_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k=1}^{2N} \mathbb{1}_{[k \neq i]} \exp(\text{sim}(z_i, z_k) / \tau)}$$

其中：
- $\text{sim}(u, v) = u^\top v / (\|u\| \|v\|)$ 为 **$\ell_2$ 归一化后的余弦相似度**
- $\tau$ 为**温度参数**（temperature）
- $\mathbb{1}_{[k \neq i]}$ 是指示函数，排除自身

最终损失为所有正样本对的双向损失之和：

$$\mathcal{L} = \frac{1}{2N} \sum_{k=1}^{N} [\ell(2k-1, 2k) + \ell(2k, 2k-1)]$$

**关键设计因素**：

| 因素 | 作用 |
|------|------|
| $\ell_2$ 归一化 | 使相似度仅取决于方向而非模长，防止"长向量"主导损失 |
| 温度参数 $\tau$ | 控制对负样本的"软硬"程度——较小的 $\tau$ 更关注最难的负样本 |
| 大 batch size（N=8192） | 每个正样本对拥有 16382 个负样本，提供充分的对比信号 |
| 无记忆库 | 简化框架，仅使用当前 batch 内的样本作为负样本 |

**反向传播**：
- 损失通过投影头 $g$ 和编码器 $f$ 反向传播
- 使用 LARS 优化器以支持大 batch size（8192）下的稳定训练
- 训练步数远多于监督学习（如 1000 epochs）

### 第五步：下游任务评估（后处理）

训练完成后：
1. **丢弃投影头 $g(\cdot)$**——仅保留基础编码器 $f(\cdot)$
2. 使用编码器输出 $h$ 作为固定特征表示
3. 在下游任务上训练线性分类器（linear evaluation）或进行微调（fine-tuning）

**主要结果**：
- **线性评估**：SimCLR 在 ImageNet 上达到 76.5% top-1 准确率，相对此前 SOTA 提升 7%
- **1% 标签微调**：仅使用 1% 的 ImageNet 标签，达到 85.8% top-5 准确率，优于使用 100 倍标签的 AlexNet
- **迁移学习**：在 12 个自然图像分类数据集中的 10 个上匹配或超越强监督基线
- **更大模型**：SimCLR (4×) 匹配了监督 ResNet-50 的性能

## 核心结论

1. **数据增强组合是决定性因素**：随机裁剪 + 颜色失真是最关键的增强组合
2. **投影头至关重要**：非线性投影使编码器保留更多信息
3. **归一化嵌入 + 温度参数**：$\ell_2$ 归一化和适当调整的 $\tau$ 对比损失有效
4. **大 batch + 长训练**：对比学习比监督学习更需要大 batch size 和更长的训练步数
5. **更宽更深的网络**：与监督学习一致，更大的模型容量带来更好的表示质量

---

> **📎 实例级 vs 像素级对比学习 — 分析注解**
>
> SimCLR 定义了一种清晰的**实例级（instance-level）对比学习**范式，其核心特征如下：
>
> **1. 对比粒度的定义**
> - 对比的**基本单元是整张图片（一个实例）**
> - 正样本对 = 同一图片的不同增强视图
> - 负样本 = batch 中其他图片的所有增强视图
> - 损失函数逐样本（per-sample）计算：每个实例在 batch 中被视为一个独立类别
>
> **2. 对比发生在哪个空间？**
> - 对比发生在**投影后的全局表示向量**（$z \in \mathbb{R}^{128}$）之间
> - 这个向量用整个特征图经过平均池化（avg pool）得到，是**全局描述符**
> - 对比的是"这张图和那张图的关系"，而非"图像内部像素/区域的关系"
>
> **3. 学习到的表示特性**
> - 学习到的是**全局语义判别性**特征（如"这是一只狗 vs 这是一只猫"）
> - 对数据增强（裁剪、颜色变化）具有不变性
> - 对实例级别的身份变化敏感
>
> **4. 与像素级对比学习的关键区别（预期）**
> - 像素级对比学习的**基本单元是像素或局部区域**——正样本来自同一图片的不同像素/区域，负样本是不同图片的像素/区域
> - 像素级方法对比发生在**特征图的每个空间位置**，而不是全局池化后的单个向量
> - 像素级方法需要维护**空间分辨率**，输出不是 $h \in \mathbb{R}^d$，而是 $\mathbf{H} \in \mathbb{R}^{C \times H \times W}$，对比在每个空间位置 $(c, h, w)$ 上进行
> - 实例级优化的是"整体辨识度"（discriminativeness of instances），像素级优化的是"局部一致性"（spatial/local consistency）
>
