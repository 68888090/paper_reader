# 对比学习：实例级优化 vs 像素级优化

## 阅读视角：聚焦粒度——从"整图判别"到"像素判别"

### 阅读动机

对比学习中存在两种根本不同的优化粒度：**实例级（instance-level）**和**像素级（pixel-level）**。通过对比 SimCLR（实例级代表）和 DenseCL（像素级代表），理解两种范式在设计哲学、技术实现和适用场景上的本质差异。

---

## 核心对比：设计哲学的根本差异

### SimCLR → 实例级优化
将每张图片视为一个**独立实例**，学习目标是区分"这张图 vs 那张图"。

### DenseCL → 像素级优化  
将特征图的每个**空间位置**视为独立对比单元，学习目标是确保同一图片不同视图中**对应位置的特征一致**。

---

## 维度一：对比的基本单元

| 维度 | 实例级（SimCLR） | 像素级（DenseCL） |
|------|-----------------|-------------------|
| 对比单元 | 整张图片（1个全局向量） | 特征图每个空间位置（$S_h \times S_w$ 个局部向量） |
| 正样本定义 | 同一图片的两个增强视图的全局表示 | 同一图片两个增强视图中**对应空间位置**的局部表示 |
| 负样本定义 | batch 中其他图片的全局表示 | 特征图中所有其他空间位置的局部表示 + 记忆库 |
| 每次迭代的对比数 | $2N$ 个对比任务（$N$ 为 batch size） | $2N \times S_h \times S_w$ 个对比任务（数量级高出 100~1000 倍） |

### 关键启示

实例级方法：batch size 越大（如 8192），负样本越丰富。  
像素级方法：负样本天然丰富（每个特征图有 $S_h \times S_w$ 个位置可作负样本），对 batch size 的依赖显著降低。

---

## 维度二：特征提取架构的差异

```
实例级（SimCLR）：
  Input → Backbone → AvgPool → h (2048-dim) → Projection Head → z (128-dim)
  对比发生在 z 这一层（单个全局向量）

像素级（DenseCL）：
  Input → Backbone → Feature Map (C×Sh×Sw) → 1×1 Conv → Q (d×Sh×Sw)
  对比发生在 Q 的每个空间位置 (h, w)
```

| 组件 | 实例级（SimCLR） | 像素级（DenseCL） |
|------|-----------------|-------------------|
| Backbone 输出 | 经过 AvgPool 的全局向量 | 保持空间分辨率的特征图 |
| 投影头 | MLP（全局池化后全连接） | $1 \times 1$ 卷积（保持空间维度） |
| 空间信息 | **被丢弃**（AvgPool 消除空间维度） | **被保留**（每个位置独立对比） |
| 输出形状 | $\mathbb{R}^d$ | $\mathbb{R}^{d \times S_h \times S_w}$ |

### 关键启示

- **AvgPool 是实例级对比的信息瓶颈**：它将 $C \times H \times W$ 的特征压缩为 $C$ 维向量，空间结构信息全部丢失
- **像素级对比通过 $1 \times 1$ 卷积保留空间结构**：每个空间位置的对比信号迫使 backbone 在所有空间位置学习有判别力的局部特征

---

## 维度三：正样本匹配机制

```
实例级：不需要空间对应
  视图1 的整张图 ←→ 视图2 的整张图
  （No spatial correspondence needed）

像素级：需要精确的空间对应
  视图1 位置 (h,w) ←→ 视图2 位置 (h,w)
  （Requires spatial correspondence between views）
```

这是两种范式**最根本的技术差异**：

- **实例级**：正样本对是通过**语义不变性**定义的——无论图片被如何裁剪、扭曲颜色，它仍然是同一张图片
- **像素级**：正样本对是通过**空间对应性**定义的——视图 1 中 $(h, w)$ 的像素特征，其正样本是视图 2 中**同位置**的像素特征（这依赖于增强操作中保持的空间对齐关系，如相同的裁剪和 resize）

---

## 维度四：损失函数的粒度

**实例级 NT-Xent（SimCLR）**：

$$\ell_{\text{inst}} = -\log \frac{\exp(z_i \cdot z_j / \tau)}{\sum_{k=1}^{2N} \mathbb{1}_{[k \neq i]} \exp(z_i \cdot z_k / \tau)}$$

- 每个 mini-batch 计算 $2N$ 次（N 对正样本）
- $z_i$：全局向量

**像素级密集对比损失（DenseCL）**：

$$\mathcal{L}_{\text{dense}} = \frac{1}{S_h S_w} \sum_{h,w} -\log \frac{\exp(\mathbf{Q}_{h,w} \cdot \mathbf{K}^+_{h,w} / \tau)}{\sum_{k^-} \exp(\mathbf{Q}_{h,w} \cdot \mathbf{k}^- / \tau)}$$

- 每个 mini-batch 计算 $2N \times S_h \times S_w$ 次
- $\mathbf{Q}_{h,w}$：位置 $(h,w)$ 的局部特征向量
- 对比发生在**特征图坐标系**的每个格点上

### DenseCL 的混合策略

DenseCL **不是纯像素级**方法——它保留了全局损失，采用混合损失：

$$\mathcal{L} = (1 - \lambda) \underbrace{\mathcal{L}_{\text{global}}}_{\text{实例级}} + \lambda \underbrace{\mathcal{L}_{\text{dense}}}_{\text{像素级}}$$

默认 $\lambda = 0.5$。这一点非常重要——说明**实例级信号和像素级信号是互补的**，不应用一个完全替代另一个。

---

## 维度五：优化目标与学习到的特征

| 维度 | 实例级 | 像素级 |
|------|--------|--------|
| 优化目标 | 全局语义判别力 | 局部空间一致性 + 局部语义判别力 |
| 不变性学习 | 对增强变换的整体不变性 | 对增强变换的**逐位置**不变性 |
| 特征分布 | 每个实例形成一个聚类中心 | 每个语义区域/像素形成聚类 |
| 下游任务适配 | 图像分类 | 目标检测、语义分割、实例分割 |
| 需要的特征类型 | 全局描述符（what is in this image?） | 局部描述符（what is at this location?） |

---

## 维度六：实验验证的差异

| 评估维度 | 实例级（SimCLR） | 像素级（DenseCL） |
|---------|-----------------|-------------------|
| 主要评估协议 | 线性评估（ImageNet 分类 Top-1） | 密集预测微调（检测 mAP、分割 mIoU） |
| 关注的指标 | Top-1 / Top-5 分类准确率 | AP / AP50 / AP75 / mIoU |
| 下游任务 | 图像分类、迁移分类 | 目标检测、语义分割、实例分割 |
| 与监督基线的对比 | 匹配监督 ResNet-50 | 大幅超越 ImageNet 监督预训练 |

DenseCL 的核心论文动机就是：**更好的图像分类 ≠ 更好的目标检测**。因此在密集预测任务的评估上，实例级方法天然存在不足，像素级方法填补了这一空白。

---

## 总结：何时用哪种优化？

| 场景 | 推荐 | 理由 |
|------|------|------|
| 下游任务是图像分类 | 实例级（SimCLR） | AvgPool 提供的全局描述符天然适配分类 |
| 下游任务是目标检测/分割 | 像素级（DenseCL） | 保留空间结构的局部特征直接服务于密集预测 |
| 数据量少 | 像素级 | 像素级提供更多对比任务（$S_h \times S_w$ 倍），数据利用率更高 |
| 需要极简框架 | 实例级 | SimCLR 无需记忆库、无需空间对应，实现最简 |
| 两者兼顾 | 混合策略（DenseCL） | $(1-\lambda)\mathcal{L}_{\text{global}} + \lambda\mathcal{L}_{\text{dense}}$ |

### 核心理解

实例级对比学习通过 **AvgPool 这个信息瓶颈**，迫使模型学习全局语义判别力；像素级对比学习通过 **保留空间分辨率 + 逐位置对比**，迫使模型在每个局部都学得有判别力。两者的本质差异不在于损失函数的形式（都是 InfoNCE），而在于 **"对比发生在哪个空间"**——全局向量空间 vs 特征图空间坐标系。
