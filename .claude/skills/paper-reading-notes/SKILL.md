# Paper Reading Notes — 读后感处理与用户画像构建

## 核心理念

读后感的三个价值：
1. **充实论文总结** — 用户个人判断是最有价值的注释
2. **暴露真实关注点** — 读后感的关注点比提问更诚实地反映用户在意什么
3. **构建用户画像** — 长期累积，形成研究兴趣、阅读习惯、写作风格的完整画像

## When to Use

用户分享对论文的看法/感受/纠正时触发：
- "这篇文章的读后感是..."
- "读完感觉..."
- "这篇文章好像不是我想要的..."
- "我觉得这篇文章重要的是..."
- 任何用户阅读后对论文内容的主观判断和反思

## Workflow

### Step 1: 解析读后感

从读后感中提取以下信息：

**关注点提取**：
- 用户提到了论文的哪个方法组件？（如：空token、对比损失、投影头）
- 用户关心什么维度？（如：实例级vs像素级、是否可迁移、计算开销）
- 用户提出了什么疑问或纠正？

**写作风格提取**：
- 高频用词（如"本质上"、"核心在于"、"这意味着"）
- 句式特征（口语化/学术化、长短句偏好、是否喜欢用"当然""不过"）
- 表述习惯（喜欢先总结再展开？喜欢做对比？喜欢加限定条件？）

### Step 2: 追加到论文总结

遵循 `modify_summary.md` 规则，在对应的 `src/items/<theme>/paper_<id>.md` 底部以引用条形式追加：

```markdown
> **📎 读者读后补充（YYYY-MM-DD）**
>
> [用户的读后感核心内容]
```

若用户纠正了对论文的原有理解（如"这不是我要找的空查询方法"），标注为**校正型读后感**：

```markdown
> **📎 读者校正（YYYY-MM-DD）**
>
> [纠正说明]。这一条可用于后续避免同类误解。
```

### Step 3: 更新用户画像

读取 `src/profiles/user_profile.md`，更新以下区域：

**关注维度更新**：
- 若提取的关注点匹配已有维度 → 增加提及次数，追加关联论文
- 若为新关注维度 → 新增条目

**写作风格更新**：
- 对比本次读后感的用词/句式与已有画像
- 新增反复出现的新特征
- 若与已有特征冲突（如之前偏好短句现在偏好长句），以新数据为准

**阅读历史追加**：
```markdown
### YYYY-MM-DD: [论文标题]
- 读后感摘要：[一句话]
- 提取关注点：[维度1, 维度2]
- 情感倾向：[正面/反面/中性/纠正]
```

**校正记录**：
若用户纠正了对某概念的误解，单独记录在画像的"校正记录"区域。

### Step 4: 反馈回分析系统

更新完画像后，提示用户画像的关键变化：
- "你的'实例级 vs 像素级'关注维度权重已从 3 提升到 4"
- "你的写作风格画像新增特征：喜欢使用对比式表述"

后续 paper-analyst、paper-customize 执行时主动读取用户画像。

## 用户画像文件结构

`src/profiles/user_profile.md` 采用以下结构：

```markdown
# 用户画像

> 自动生成于 YYYY-MM-DD，最新更新于 YYYY-MM-DD

## 关注维度矩阵

| 维度 | 提及次数 | 首次关注 | 最近关注 | 关联论文 |
|------|----------|----------|----------|----------|
| 实例级 vs 像素级对比 | 3 | 2026-05-05 | 2026-05-05 | SimCLR, DenseCL |
| 空查询机制 | 1 | 2026-05-05 | 2026-05-05 | Classifier-Free Guidance |

## 研究兴趣演变

（按时间线记录关注点的变化）

## 方法论偏好

- [从读后感中推断的方法偏好，如：偏好像素级密集方法、关注方法实用性]

## 写作风格画像

### 常用表达
- [高频词/短语，如："本质上"、"核心在于"、"重要的事是..."]

### 语气偏好
- [学术严谨 / 白话通俗 / 口语化 / 类比丰富]

### 句式特征
- [长短句偏好，是否喜欢列表，是否喜欢加限定词]

### 修改模式
- [常对总结做的修改类型，如：补充对比视角、缩短描述、加"为什么重要"]

## 校正记录

（记录用户纠正过的理解偏差，避免重复误解）

## 阅读历史

### YYYY-MM-DD: [论文标题]
- 读后感摘要：[...]
- 提取关注点：[...]
- 情感倾向：[...]
```

## 画像更新规则

1. **增量不覆盖**：每次更新保留历史，通过提及次数跟踪关注度变化
2. **维度进化**：同一维度反复出现 → 提升权重，主题文件夹中优先排序
3. **校正优先级最高**：用户明确纠正的理解 > 其他所有信号
4. **风格缓慢收敛**：写作风格在 3-5 篇读后感后才能稳定，初期每次更新变化较大

## 与其他 skill 的集成

| skill | 如何使用画像 |
|-------|------------|
| **paper-analyst** | 生成总结前读取写作风格，应用用户的表达习惯和侧重点 |
| **paper-customize** | 建立视角时优先推荐用户画像中高权重的关注维度 |
| **paper-contrast** | 对比时检查校正记录，避免重复用户已纠正过的误解 |

## 初始化

首次使用时执行：
```bash
python3 .claude/skills/paper-reading-notes/scripts/init_profile.py
```

该脚本创建 `src/profiles/user_profile.md` 的基本模板。之后每次读后感处理时自动更新。
